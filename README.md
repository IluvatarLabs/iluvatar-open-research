# Iluvatar Open Research Initiative

Open source science powered by [Marvin](https://iluvatarlabs.github.io/marvin/). We run research projects on public data, publish every iteration openly, and invite the field to critique, extend, and validate.

Learn more at [iluvatarlabs.github.io/iori](https://iluvatarlabs.github.io/iori/).

**Code**: Apache 2.0 · **Text and figures**: CC-BY 4.0 · **IP on findings**: none

## Contribute

Four ways to participate. See [CONTRIBUTING.md](CONTRIBUTING.md) for details.

| Action | How |
|--------|-----|
| **Propose a project** | [Submit a question](https://github.com/IluvatarLabs/iluvatar-open-research/issues/new?template=new-project-idea.yml) |
| **Review an active project** | [Open a critique](https://github.com/IluvatarLabs/iluvatar-open-research/issues/new?template=scientific-critique.yml) |
| **Share a resource** | [Suggest a dataset](https://github.com/IluvatarLabs/iluvatar-open-research/issues/new?template=new-data-source.yml) or [suggest a paper](https://github.com/IluvatarLabs/iluvatar-open-research/issues/new?template=literature-suggestion.yml) |

Have a question or want to brainstorm before filing an issue? Start a [Discussion](https://github.com/IluvatarLabs/iluvatar-open-research/discussions).

## How updates work

1. Marvin runs a new analysis iteration.
2. Internal PhDs review and vet the outputs.
3. A pull request is opened with a structured summary: what changed, new datasets, new or revised hypotheses, retractions.
4. The PR sits open for **7 days** for community review.
5. After the review period, the PR merges. The project manifest and hypothesis cards are updated.
